# excel_to_md.py — Excel方眼紙・管理会計PL変換ツール

## 概要

`excel_to_md.py` はExcelの「意味のある構造」をMarkdown / JSONに変換するPythonスクリプト。
変換の基本方針は **「値・階層・意味を保持し、レイアウトの再現は二次的」**。

-----

## インストール

```bash
pip install openpyxl
```

-----

## 使い方

```bash
# Markdown出力（デフォルト）
python excel_to_md.py input.xlsx output.md

# JSON出力
python excel_to_md.py input.xlsx output.json --format json

# シート指定
python excel_to_md.py input.xlsx --sheet "損益計算書"

# 標準出力
python excel_to_md.py input.xlsx
```

-----

## 変換ロジック（パターン別）

### 1. 管理会計PL・予算表（列オフセット字下げ）

**Excelの表現:**  
A列=大項目、B列=中項目、C列=小項目という「列をずらすことで階層を表現」するパターン。

**検出方法:**

- 数値セルが全体の25%以上
- 行ヘッダーが文字列、値列が数値

**変換結果:**  
Markdownテーブル。空列がインデントを表現する。

```markdown
| 科目       |            |                  | 2023年度 | 2024年度 |
| 売上高     |            |                  |          |          |
| ---------- | ---------- | ---------------- | -------- | -------- |
|            | 製品A売上  |                  | 5000     | 6000     |
|            | 製品B売上  |                  | 3000     | 3500     |
| 売上高合計 |            |                  | 10000    | 11700    |
```

> **プログラムが読む際の注意:** 空列の数 = インデントレベル（0始まり）。  
> `col_index_of_first_nonempty_in_row - min_col_of_section = indent_level`

-----

### 2. フローチャート・業務フロー図（矢印+方眼紙）

**Excelの表現:**  
`→` `↓` `↑` などの記号を方眼紙セルに配置して流れを表現。

**検出方法:**

- 矢印文字を含む
- 数値セルが25%未満

**変換結果:**  
固定幅コードブロック。セルの位置関係を空白で再現。

```markdown
```
   受注        → 在庫確認    → 出荷指示
   ↓             ↓
   与信確認       在庫なし    → 発注
```
```

> **注意:** 図形の意味（分岐条件など）はセル値として復元されないため、  
> LLMによる後処理で構造を解釈することを推奨。

-----

### 3. クロス集計表・マトリクス

**Excelの表現:**  
行と列の両方に見出しがあるクロス表。セル結合で見出しをまとめている場合も多い。

**検出方法:**

- 1行目が文字列のみ（数値なし）= ヘッダー行
- 以降は数値データ

**変換結果:**  
標準Markdownテーブル。

```markdown
| 製品＼地域 | 東日本 | 西日本 | 海外 | 合計 |
| ---------- | ------ | ------ | ---- | ---- |
| 製品A      | 1200   | 800    | 400  | 2400 |
| 合計       | 2100   | 1900   | 3300 | 7300 |
```

-----

### 4. セクション分割

**Excelの表現:**  
1枚のシートに複数の表が縦に並んでいる、結合セルが見出しになっている。

**検出方法:**

- 2行以上の空行 = セクション境界
- 横幅の半分以上を占める結合セル + 太字 = セクション見出し

**変換結果:**  
各セクションを `## 見出し名` で区切ったMarkdown。

-----

## 変換できないもの / 限界

|パターン       |対応状況               |理由・代替                 |
|-----------|-------------------|----------------------|
|罫線で描いた組織図  |△コードブロック           |階層の意味は復元不可。LLM後処理推奨   |
|画像・グラフ     |✗スキップ              |openpyxlは画像データを文字化できない|
|複雑な結合(縦横両方)|△値のみ保持             |結合形状の意味は失われる          |
|計算式        |△data_only=Trueで値取得|数式ロジックは別途解析が必要        |
|条件付き書式     |✗無視                |色や強調の意味は失われる          |
|複数行ヘッダー    |△1行目のみ             |2段組み見出しは結合で対処         |
|フォントによる意味付け|△太字のみ考慮            |色・サイズの意味は失われる         |

## JSON出力の構造

```json
{
  "シート名": {
    "sections": [
      {
        "title": "セクション見出し（あれば）",
        "rows": [
          ["科目", "", "", "2023年度", "2024年度"],
          ["売上高", "", "", "", ""],
          ["", "製品A売上", "", "5000", "6000"]
        ]
      }
    ],
    "raw": {
      "rows": [
        [{"value": "科目", "bold": true}, {"value": "2023年度", "bold": true}],
        [{"value": "売上高", "bold": true}],
        [{"value": "製品A売上"}, {"value": "5000"}]
      ]
    }
  }
}
```

`sections` は整形済み（空列削除済み）、`raw` は元の構造（bold/indent/span情報付き）。

## PLのMarkdown → 構造化データへの変換例（後処理）

変換後のMarkdownをさらにプログラムで処理する場合:

```python
import re

def parse_pl_markdown(md: str) -> list[dict]:
    rows = []
    for line in md.splitlines():
        if not line.startswith("|") or line.startswith("| -"):
            continue
        cells = [c.strip() for c in line.strip("|").split("|")]
        # インデントレベル = 先頭から最初の非空セルのindex
        indent = next((i for i, c in enumerate(cells) if c), 0)
        label = cells[indent] if indent < len(cells) else ""
        values = [c for c in cells[indent+1:] if c]
        rows.append({"indent": indent, "label": label, "values": values})
    return rows
```
